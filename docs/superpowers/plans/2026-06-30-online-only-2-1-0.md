# Online-Only Workspace v2.1.0-test Implementation Plan

> **For agentic workers:** Execute tasks in order. Contract interfaces in Task 1 are frozen before parallel Tasks 2–4. Track every step with the checkboxes below.

**Goal:** Replace the local-first workspace with server-authoritative reads and transactional online writes, remove persistent local business data, and deliver a verified fixed-signature `2.1.0-test` APK.

**Architecture:** Shared Zod contracts define Workspace queries, commands, revisions, idempotency, and temporary assets. Fastify services operate directly on PostgreSQL and storage; React keeps only current-session data and local drafts in memory. After cutover, legacy Dexie/sync/cache code is deleted and a small idempotent purge removes data left by older installations.

**Tech Stack:** Next.js, React, TypeScript, Zod, Fastify, Drizzle/PostgreSQL, Capacitor, Playwright, Android/Gradle.

## Global Constraints

- Server is the only authoritative source for business entities and images.
- No business entity, image, draft, or diagnostic event may be persisted locally.
- Auth session, MiniMax key, and the non-business purge marker remain under their existing storage rules.
- A write succeeds only after the server transaction commits and the client re-reads the entity.
- Every mutation has `clientMutationId`; updates/deletes also have `expectedRevision`.
- No optimistic updates, outbox, background sync, or hidden local fallback.
- All source deletions use the project trash workflow after before/after `git status --short` checks.
- Final version is `2.1.0-test`; APK uses the fixed `CN=fangzheng` signing key.
