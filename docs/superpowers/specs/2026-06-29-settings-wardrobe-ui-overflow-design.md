# 设置页与衣橱首页 UI 溢出修复设计

## 范围

本轮只修复 UI，不修改云同步数据、不猜测或覆盖已丢失的衣橱名称。

## 问题与修复

1. 设置页和衣橱首页的单列 Grid 使用内容最小宽度，长名称会撑大整列。将页面主 Grid 改为可压缩的 `minmax(0, 1fr)` 列，并在顶部操作行和名称容器补齐 `min-width: 0`，长名称保持单行省略。
2. 设置页每个模块复用 `.surface` 阴影。仅在设置首页移除直接子卡片阴影，不改变其他页面的全局卡片样式。
3. 多层底部弹窗会重复覆盖滚动锁的原始样式。滚动锁只在计数从 0 变为 1 时保存和施加，最后一层关闭时恢复。
4. 删除当前未提交的页面级 `overflow-hidden` 硬裁切方案；它不解决内部宽度，也会增加滚动风险。

## 验证

- 新增最小回归检查，覆盖可压缩 Grid、长名称省略、设置卡片无阴影和嵌套滚动锁。
- 运行 `npm run typecheck`、相关逻辑检查和 `npm run build`。
- 以手机窄屏检查设置页和衣橱首页；长名称不得改变卡片或瀑布流宽度，页面必须可继续纵向滚动。

## 不在本轮处理

- 云端衣橱记录 `payload` 为空、名称/简介/`dexieId` 丢失。
- 已丢失衣橱名称的数据恢复或修复策略。
