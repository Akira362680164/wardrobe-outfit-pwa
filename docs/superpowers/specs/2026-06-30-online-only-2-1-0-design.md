# 纯线上工作区 v2.1.0-test 设计

## 目标

将衣橱 App 改为服务器唯一权威数据源。客户端只在当前页面内存中处理选图、裁切、缩略图、AI 识别结果和未提交表单；正式业务数据、图片和诊断工单均由服务器保存。升级完成后物理删除本地优先同步、业务数据库、持久图片缓存和相关旧测试。

## 已确认产品规则

- 正式数据只以服务器结果为准，不做乐观更新。
- 单品、衣橱位置、套装、套装关联、种草、旅行计划、每日计划、穿着记录、收藏、图片和试穿档案不得持久化为本地业务缓存。
- MiniMax Key、登录会话和不含业务内容的迁移完成标记可以继续使用现有安全边界保存。
- 诊断事件仅保存在当前进程的有界内存缓冲；用户主动提交后上传服务器，不写 IndexedDB、localStorage、Cache Storage 或文件系统。
- App 被关闭或系统杀死后，未提交草稿和尚未上传的诊断事件允许丢失。
- 旧本地业务代码和失效测试物理删除，不保留隐藏回退路径。
- 最终 Android 版本为 `2.1.0-test`，使用固定 `CN=fangzheng` 签名。

## 架构

### 共享契约

`packages/cloud-contracts` 增加 Workspace 查询、业务命令、临时资产和统一错误契约。所有写请求包含 `clientMutationId`，更新和删除包含 `expectedRevision`。业务响应返回最终实体、revision、图片引用和 requestId，不返回 DataURL。

### 服务端

`services/wardrobe-api` 新增 `/api/workspace/**` 查询和命令路由。数据库继续使用现有业务表，并扩展资产和幂等记录：

- 临时资产在实体创建前归当前用户和 `clientMutationId` 所有。
- 原图和缩略图上传完成后才能提交业务命令。
- 命令事务校验资产、创建或更新业务实体、绑定资产、更新 revision 并保存完整幂等响应。
- 同一用户重复提交同一 `clientMutationId` 返回第一次提交的结果。
- 未绑定临时资产按过期时间清理。
- 删除、种草转衣橱、撤销购买、标记已穿和级联关系更新在单个数据库事务完成。

### 客户端

客户端复用现有鉴权、Capacitor/fetch 双通道、requestId 和脱敏诊断能力，新增薄的线上请求层与按业务域组织的 Repository。页面数据保存在 React 内存中：

- 首次加载：无旧数据，显示启动页或页面骨架。
- 刷新：保留当前会话数据，失败显示轻量行内提示。
- 详情：每次进入都重新读取服务器。
- 图片：下载后只保留当前会话 Object URL；退出账号或卸载页面时回收。
- 写入：等待服务器成功并重新读取后才更新正式页面状态。

## 图片与 AI 数据流

1. 用户在本地选择原图。
2. 客户端在内存中保存原图，进行裁切并记录归一化裁切框。
3. 客户端从原图和裁切框生成缩略图。
4. 客户端按用户动作发起 MiniMax 识别，在内存中保存结构化属性。
5. 第一次提交生成 `clientMutationId` 和临时资产会话。
6. 客户端上传原图与缩略图；服务端校验 MIME、大小、SHA-256、宽高和用户归属。
7. 客户端提交属性、AI 结果、裁切框和临时资产 ID。
8. 服务端事务绑定资产并提交最终实体。
9. 客户端重新读取该实体；可读后显示成功并刷新上级列表。

失败时停留当前页面、保留内存草稿并恢复按钮。草稿未变化的重试复用同一 `clientMutationId`；任一业务字段、图片或裁切框变化后生成新 ID。

## API 范围

查询覆盖：overview、garments、outfits、wishlist、locations、trip plans、outfit plans、packing checklist、wear summary 和 try-on profile。

命令覆盖：

- 单品批量创建、编辑、重裁、删除、标记已穿和取消已穿。
- 种草创建、编辑、删除、转入衣橱和撤销购买。
- 套装创建、编辑、删除、收藏、取消收藏和实拍图。
- 衣橱位置创建、编辑和删除。
- 旅行计划、每日计划、打包清单的创建、编辑和删除。
- 标记计划已穿和取消已穿。
- 试穿档案及参考照片保存。
- 临时资产会话、上传、状态查询和放弃。

所有查询按 JWT 用户隔离；客户端不得传入 userId。列表稳定排序并支持分页，计划支持日期范围。

## 状态与 UI

列表显式区分 `loading`、`ready`、`refreshing`、`error` 和 `refresh_error`；详情显式区分加载、保存、删除、冲突、未找到和失败。首次加载不得显示空状态，普通写入不得使用全屏遮罩。

共享 UI 复用现有颜色、`lucide-react`、`MotionToast` 和录入页行内提示样式，提供附件指定的加载页、错误页、骨架、按钮 Spinner、图片占位和写入保护组件。错误提示保持浅色行内样式，按钮与返回键在 Android 窄屏和横屏可用。

## 旧链路删除与升级清理

线上读写全部验收后，逐项移入系统废纸篓并提交其 Git 删除：

- Account Workspace Dexie、Repository 和 Registry 的业务数据能力。
- Outbox、Bootstrap、Push、Pull、冲突处理和后台同步。
- `workspace-write-commands` 与页面运行时 `bridge*`。
- 本地图片持久缓存、资产恢复、缩略图回填和依赖本地正式数据的备份功能。
- 旧 Sync 客户端、共享契约、服务端路由及只验证旧架构的测试。

新增独立的一次性升级清理器：在用户成功登录且线上 Overview 可读后，删除已知 Workspace IndexedDB、旧业务 Cache Storage 和旧 Workspace 注册信息；全部成功后写入不含业务数据的 `online-only-purge-v1` 标记。清理失败不得阻止线上数据使用，但必须记录内存诊断事件并在下次启动重试。不得读取或上传旧本地业务内容。

## 部署顺序

1. 部署兼容数据库迁移、临时资产和 Workspace API。
2. 验证服务端查询、事务、幂等和资产清理。
3. 发布纯线上客户端并完成浏览器与 Android 验收。
4. 确认新客户端不再调用旧 Sync 后，从当前源码物理删除旧链路。
5. 升级 `2.1.0-test`，构建固定签名 APK。

本次测试服务允许旧 APK 在服务端旧路由删除后停止工作；不为旧 APK 保留长期兼容接口。

## 并行边界

共享契约冻结后允许三个 Subagent 并行：服务端 Workspace API、客户端线上读取与状态 UI、客户端线上写入与录入状态。主 Agent 负责契约、数据库迁移审核、集成、物理删除、全量验证、版本历史和 APK。

## 验收

- 静态搜索证明正式页面不再导入 Dexie、Outbox、Sync Engine、Workspace Write Commands、持久图片缓存或运行时 Bridge。
- 服务端测试证明用户隔离、revision 冲突、幂等重放、事务回滚和临时资产清理。
- 客户端测试证明首次加载、刷新失败保留会话数据、失败保留草稿、无乐观更新和图片局部重试。
- Playwright 覆盖主要读写、图片失败、超时、500、JWT 失效和幂等重试。
- Android 模拟器覆盖冷启动、断网恢复、相机/相册、裁切、AI、上传失败重试、返回键、横竖屏和卸载重装。
- `typecheck`、共享契约、API、逻辑、build、E2E、review gate 和 `android:apk` 全部通过。
- APK 为 `衣橱穿搭助手-v2.1.0-test.apk`，包名和固定签名核验通过。
