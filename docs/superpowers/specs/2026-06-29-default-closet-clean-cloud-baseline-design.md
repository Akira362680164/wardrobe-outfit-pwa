# 默认衣橱与云端数据重置方案

## 要解决什么

目前衣橱可以被全部删除，默认衣橱也可被删除和修改，造成衣物无处存放、系统反复补建衣橱和同步数据混乱。

本次目标是从规则上结束这种混乱，并清空当前线上测试数据，让所有账号从全新状态开始。

## 默认衣橱规则

- 新账号创建后，自动拥有一个“默认衣橱”。
- 默认衣橱的名称和简介都固定为“默认衣橱”。
- 用户不能改名、修改简介或删除默认衣橱。
- 设置页中的默认衣橱只用于展示，点击后不进入编辑页。
- 删除其他衣橱时，用户始终可以把衣物迁移到默认衣橱。
- 设置页不再提供“清空数据”入口，避免用户在不清楚影响范围时删除整个 App 数据。
- 用户仍可在对应功能中单独删除衣物、自定义衣橱或其他内容。

## 无网和同步规则

- 默认衣橱先在手机本地创建，不需要等待网络。
- 没网时仍可正常进入 App、录入衣物和管理衣橱。
- 恢复网络后，App 在后台把本地数据继续同步到云端。
- 衣橱同步必须完整保留名称、简介和排序，不能再出现重装后只恢复空衣橱的情况。

## 线上数据重置

本次会永久清空当前测试服务器 `111.231.98.86` 中的：

- 所有注册账号、登录状态和认证数据。
- 所有衣橱、衣物、套装、种草、计划和个人画像数据。
- 所有上传的衣物图片、试穿照片和诊断文件。
- 所有历史同步记录和远程诊断记录。

不删除服务器程序、数据库结构、密钥或部署配置。清空后旧账号无法再登录，需要重新注册。

## 执行顺序

1. 先修改 App 和云端同步规则。
2. 运行本地测试，确认默认衣橱不能被修改或删除。
3. 更新云端服务。
4. 在真正执行永久删除前，再次向用户展示要删除的服务器和数据范围并获得确认。
5. 清空线上数据，重启并检查服务状态。
6. 重新注册一个全新账号，完整验证默认衣橱的本地使用和云端同步。

## 验收标准

- 全新账号第一次进入 App 就能看到默认衣橱。
- 默认衣橱没有改名、编辑和删除入口。
- 设置页不再显示“清空数据”。
- 断网时可以录入衣物，联网后可自动同步。
- 清空本地数据并重新登录后，默认衣橱的名称、简介和衣物归属保持正确。
- 删除自定义衣橱时，可把衣物迁移到默认衣橱。

## 重要风险

线上数据清空后不可恢复，且按本次要求不保留旧数据备份。清空前会再次进行最终确认。
